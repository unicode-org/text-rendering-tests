AVAR–1

The font for this test case is a variation font with a single axis. While the ‘gvar’ table does a regular interpolation without any special quirks, the font’s ‘avar’ tables defines a sawtooth mapping. A conformant rendering engine should produce the test glyph in its maximally thin shape at axis value 100. The resulting glyph should become bolder until 250, then decrease its weight again until to 650, then become bolder again.

KERN–1

The font for this test case contains a ‘kern’ table of format 0, which in turns contains a single format 0 subtable. Without kerning, the glyphs for T, u and the dotless ı will look like normal letters. However, if the OpenType implementation correctly applies the font’s kerning data, the shapes will join to form a geometric pattern. If you can still discern the original letters ıTuTuTı in the rendering, it’s a sign that the implementation fails to correctly handle kerning.

KERN–2

Although kerning is typically reducing space between letter pairs, it’s entirely valid for a font to increase it. This test verifies that an OpenType implementation can handle such kerning tables. If the implementation works correctly, you should see four groups separated by whitespace.
